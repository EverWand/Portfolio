    Tim Sort Vs. Gnome Sort Analysis

    C++

    CSC382

    Thumbnail

    In this program, I pit against two different sorting Algorithms of Sorting Algorithms, curious as to how they would compare when handling different sized lists to sort through. I have constructed both a Gnome Sort and a Tim Sort algorithm that takes in randomly generated lists of values. I ran tests to see just how fast each sorting algorithm was when processing each list. With Tim Sort incorporating both the Insertion Sort and Merge Sort algorithms, I was able to gather more insight than just the two algorithms I began my focus on.   • Bleak Manor

    C#

    CSC245

    Programmer, Porting

    Bleak Manor was created when I was doing the Game-A-Week course. This Game was more unique than the other games we produced during the class as we all got to work together as a team to create content for the AR Board, Tilt5. The game was a 3-player asymmetric ghost hunting game that projected from the board for the players wearing the Tilt5’s Glasses, allowing the user’s to see their own perspectives while looking at the same board. Two players took the role as ghost hunters that wandered Bleak Manor through a 3rd-person perspective while the third player tried to hide around the manor as a ghost in order not be caught, seeing the environment in a top-down perspective. We used the AR projections to give the game depth so that the players felt like they were moving around in a 3D space and the Ghost got to see the world popping out towards them as they could look over walls. And gain a bird's-eye-view for any hiding places or avoid the hunters.

    Rust Pong

    Rust

    SPT323

    This was when I tried to use a much different game engine than game developers would use. Bevy is a game engine which utilizes the Rust programming language. There are bountiful amounts of differences on how you would make a game using Bevy. The two main significances are how there’s no interface for the engine, so everything must be done inside of an IDE through documents. The second and what was the more difficult thing to wrap my mind around at the time was how Bevy doesn’t use Object Oriented Programming Principles (OOP) you’d likely see for most game engines and languages. Instead, Bevy opted for the principles called the Entity-Component System (ECS), taking advantage of how you’d structure code within Rust. To understand the basics of Bevy and this different style of engine, I made a simple Pong game using entities, components, and queries to make a movement controller, collision, and bouncing logic from scratch.

    PDS300, PDS400

    Programmer, Lead Programmer

    The Cursed Men was one of the first larger teams I’ve been on, consisting of around fifteen other developers who put their efforts towards the project, all with their own wealth of knowledge and hard work when it came to their specific principles and roles on the team. I imagine myself no different, stepping up with my experience leading small teams prior to the project, I became the person in charge of implementing the UI and the lead programmer. We would have two meetings a week where we discussed our stand-ups and how to tackle problems. One of these meetings would be to make a build of where the project’s progress was so we can analyze where we were at and what possibly needed to be fixed or changed. This build-review provided an outlet to not only be critical about what needed to happen, but served as a way to celebrate our efforts and see the project grow each week. I would often be on stand-by to provide help and ensure we made progress for each week’s build, while also handling my own UI tasks.
