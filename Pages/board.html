    Rust Pong

    Rust

    SPT323

    I tried to use a different kind of game engine than what you'd expect. Bevy is a game engine which utilizes the Rust programming language. There are bountiful amounts of differences on how you would make a game using Bevy. The two main significances are how there’s no interface for the engine, so everything must be done inside of an IDE through documents. The second and what was the more difficult thing to wrap my mind around at the time was how Bevy doesn’t use Object Oriented Programming Principles (OOP) you’d likely see for most game engines and languages. Instead, Bevy opted for the principles called the Entity-Component System (ECS), taking advantage of how you’d structure code within Rust. To understand the basics of Bevy and this different style of engine, I made a simple Pong game using entities, components, and queries to make a movement controller, collision, and bouncing logic from scratch.
